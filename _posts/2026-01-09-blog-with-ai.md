# AI와 함께 블로그 만들기

**비전공자 코딩 입문자가 AI 페어 프로그래밍으로 블로그를 만든 이야기**

---

## 🤔 시작은 단순한 호기심

코딩을 막 배우기 시작한 비전공자로서, 나만의 개발 블로그를 갖고 싶었다.  
근데 문제는... **뭘 어떻게 시작해야 할지 모르겠다는 것.**

그래서 AI에게 물어봤다.

---

## 💬 실제로 입력한 프롬프트들

### 1단계: 블로그 뼈대 만들기

```
"내 GitHub Pages 블로그를 만들어줘. Jekyll 사용하고 싶어."
```

AI는 `_config.yml`, `index.html`, `_layouts/` 폴더 구조까지 알아서 만들어줬다.  
나는 그냥 **승인 버튼만 눌렀다.**

### 2단계: 디자인 요청

```
"디자인을 더 예쁘게 해줘. 깔끔하고 모던한 느낌으로."
```

AI가 CSS를 작성하고, 카드형 레이아웃을 만들고, 호버 효과까지 넣어줬다.  
**나는 결과물을 보고 "좋아" 또는 "이건 바꿔줘"라고만 했다.**

### 3단계: 카테고리 구조 설계

```
"카테고리를 4개로 나누고 싶어. 
Bridge - 자기소개
Insight - 공부 기록  
Lab - 미니 프로젝트
Log - 개발 일지"
```

AI가 각 카테고리 페이지를 만들고, 네비게이션도 연결해줬다.

### 4단계: 첫 포스트 작성

```
"Lab 카테고리에 숫자 맞추기 게임 포스트를 작성해줘."
```

게임 코드와 설명까지 포함된 포스트가 뚝딱 완성됐다.

---

## 🎯 AI 활용 팁 (입문자 관점)

| 이렇게 말하면 | 결과 |
|-------------|------|
| "블로그 만들어줘" | 기본 구조 생성 |
| "더 예쁘게" | 디자인 개선 |
| "이건 마음에 안 들어" | 수정 제안 |
| "왜 안 돼?" | 원인 분석 + 해결 |

**핵심은 구체적으로 말할수록 좋다는 것!**

```
❌ "블로그 좀 고쳐줘"
✅ "홈페이지 카드 디자인에 그림자 효과 넣어줘"
```

---

## 📚 이 과정에서 배운 것

솔직히, 처음엔 AI가 만들어주는 코드가 뭔지 하나도 몰랐다.

하지만 계속 질문하다 보니 알게 된 것들:

- **Jekyll** = 정적 사이트 생성기
- **GitHub Pages** = 무료 호스팅
- **Markdown** = 글 쓰는 문법
- **CSS** = 디자인 담당
- **permalink** = URL 규칙

**코드를 다 이해 못해도, 블로그는 만들 수 있다.**  
**그리고 만들면서 조금씩 배우게 된다.**
